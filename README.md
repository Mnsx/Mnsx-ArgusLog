# Mnsx-ArgusLog
### 项目简介

Mnsx-ArgusLog是一个专为极高并发场景打造的**工业级异步日志系统**

作为**Mnsx-Olympus**（分布式RPC框架）的核心组件之一，ArgusLog旨在为底层的Reactor网络库（AchillesLink）和高性能计算线程池（KrakenPool）提供零阻塞、低延迟、绝对时序的日志记录能力，确保业务线程的吞吐量不被缓慢的磁盘I/O所拖累

### 核心特性

* **多线程安全与解耦**
  * 基于**单例模式**的`LogHandlerManager`统一调度
  * 采用**生产者-消费者模型**，前端业务线程仅负责组装`LogEvent`并极速推入内存队列，真正耗时的落盘操作由后台专属线程异步执行
* **灵活的多目标分发**
  * 抽象`ILogHandler`接口，支持轻松挂载多个输出端，**目前内置`FileLoghandler`和`ConsoleLogHandler`**
* **优雅的RAII流式调用**
  * 提供类似`glog`的流式宏接口，利用临时对象的生命周期实现无感知的日志收集与自动触发发送
* **优雅退出机制**
  * 系统销毁时，后台线程将保证内存队列中剩余的所有日志安全落盘后再结束运行，绝不丢失关键现场数据

* **极致的前端性能**
  * 利用编译期宏（`__FILE__`）与纯指针偏移实现O(1)复杂度的相对路径截断，彻底消灭热点路径上的字符串堆内存分配

### 展望未来

* 日志滚动/切割
* 双缓冲机制

### 编译与运行

本项目使用CMake构建，可以通过以下命令运行基础测试Demo：

```c++
mkdir build && cd build
cmake ..
make
./ArgusDemo
```
